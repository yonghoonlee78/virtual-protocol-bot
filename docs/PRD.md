## 슬라이드 2: 왜 이 프로젝트인가?
### 💡 프로젝트의 필요성

**현재 문제점**
- AI 에이전트 정보가 여러 플랫폼에 분산
- 실시간 모니터링 도구 부재
- 복잡한 온체인 데이터 접근
- 통합된 분석 플랫폼 없음

**우리의 해결책**
- 중앙화된 정보 허브 구축
- 24/7 자동 모니터링 시스템
- 직관적인 대시보드 제공
- 자동 트랜잭션 실행 (거래 준비까지)

---

## 슬라이드 3: 기술 스택
### 🛠 100% 무료 기술 구성

**Backend**
- **Render.com**: 백엔드 호스팅 (무료)
- **Node.js + Express**: 서버 프레임워크
- **Socket.IO**: 실시간 통신

**Database**
- **MongoDB Atlas**: NoSQL 데이터베이스 (512MB 무료)
- 유연한 스키마 구조
- 실시간 Change Streams

**Frontend**  
- **Vercel**: 프론트엔드 호스팅 (무료)
- **React**: UI 프레임워크
- **Tailwind CSS**: 스타일링

---

## 슬라이드 4: 시스템 아키텍처
### 🏗 전체 구조

```
사용자 인터페이스
    ↓
웹 대시보드 (React) + 텔레그램 봇
    ↓
REST API + WebSocket
    ↓
비즈니스 로직 계층
    ↓
MongoDB 데이터베이스
    ↓
블록체인 (Base Network)
```

**주요 모듈**
- 모니터링 서비스
- 데이터 수집기
- 알림 시스템
- 트랜잭션 빌더

## 슬라이드 5: MongoDB 데이터 모델
### 📊 유연한 NoSQL 구조

**Agent Collection**
- 기본 정보 (address, name, symbol)
- 동적 메타데이터 (Mixed Type)
- 가격 데이터 (임베디드)
- 소셜 지표
- 거래 통계

**User Collection**
- 플랫폼별 식별자
- 포트폴리오 (임베디드)
- 워치리스트
- 설정 및 알림 정보

**Trade Collection**
- 거래 내역
- 트랜잭션 정보
- 상태 추적
